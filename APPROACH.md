# Custom Marker Detection — Technical Approach

## 1. Marker Design

### Structure
A **5×5 grid** enclosed in a thick black border on a white background.

```
┌──────────────────────────┐  ← white margin (50px)
│ ████████████████████████ │  ← black border (20px)
│ █ [OR][D1][D2][D3][D4] █ │  ← Row 0: orientation cell + 4 data cells
│ █ [D5][D6][D7][D8][D9] █ │  ← Row 1: 5 data cells
│ █ [..][..][..][..][..] █ │  ← Rows 2–4: 5 data cells each
│ ████████████████████████ │  ← black border (20px)
└──────────────────────────┘  ← white margin (50px)
```

### Orientation Cell [0,0]
A diagonal triangle: pixels where **col > row → BLACK**, else **WHITE**.
This creates a recognisable asymmetric pattern that survives all 4 rotations.

```
┌─────────┐
│ · · ███ │  ← top-right = black
│ · ▪ ███ │
│ · · · · │  ← bottom-left = white
└─────────┘
```

### Data Encoding
- 24 data cells (5×5 grid − 1 orientation cell)
- Each cell = 1 binary bit: **black = 1, white = 0**
- Maximum 9 black cells (≤ 9/24 = 37.5% black → ≥ 62.5% white, satisfying the 60% rule)
- Encodes up to 2^24 = 16.7M unique IDs

## 2. Detection Pipeline (Kotlin + OpenCV)

The `MarkerDetectorModule.kt` implements an 11-step pipeline:

### Step 1 — Decode Frame
Base64 JPEG from `react-native-vision-camera` → OpenCV `Mat`

### Step 2 — Grayscale + Blur
Convert BGR→Gray, apply 5×5 Gaussian blur to reduce sensor noise.

### Step 3 — Adaptive Threshold (×4 configs)
`ADAPTIVE_THRESH_GAUSSIAN_C + THRESH_BINARY_INV` with block sizes 11/15/21/7.
Multiple configs handle varying lighting and print quality.

### Step 4 — Morphological Close
Closes broken lines in the marker border using a 3–5px rectangular kernel.

### Step 5 — Contour Detection
`findContours` with both `RETR_EXTERNAL` and `RETR_LIST` modes.
Filters contours by:
- Minimum area (500px²)
- `approxPolyDP` → must have exactly 4 corners
- Aspect ratio ≤ 1.5 (square-ish)

### Step 6 — Perspective Warp
Corners ordered TL→TR→BR→BL, then **inset 15% toward centroid** to skip the thick black border and warp only the inner 5×5 grid.
Destination: 300×300px.

### Step 7 — Otsu Threshold
Binarises the warped patch using automatic Otsu threshold.

### Step 8 — Orientation Detection
Samples two regions in the top-left cell for each of 4 rotations:
- **TR region** (rows h/4…3h/4, cols 3w/4…w) → should be BLACK (low mean)
- **BL region** (rows 3h/4…h, cols 0…w/4) → should be WHITE (high mean)

Selects the rotation with the **largest (BL−TR) difference** (relative, not absolute — works across all lighting conditions).

### Step 9 — Grid Decode
For each of the 24 data cells (skipping [0,0]):
- Sample centre 50% of cell (avoids grid-line contamination)
- `mean < 127 → bit "1"`, else `bit "0"`

### Step 10 — Validation
- Reject if > 9 black bits (violates 60% white rule)
- Reject if all-zero (blank frame)
- Reject if ID > 20 (our specific marker set)

### Step 11 — Encode Result
Resize aligned binary image to **300×300px**, encode as PNG Base64.

## 3. Performance Optimisations

| Technique | Effect |
|-----------|--------|
| 4 threshold configs tried in parallel-order | Handles any lighting without retries |
| `RETR_EXTERNAL` tried first | Usually sufficient, avoids nested contour noise |
| All square candidates tried (not just largest) | Catches marker when other shapes are in frame |
| Capture every 400ms | 2.5 fps detection rate |
| Background thread execution | No UI blocking |

## 4. Camera Configuration

- Resolution: ~2500×2500px (set via `useCameraFormat`)
- `takePhoto({ flash: 'off' })` for consistent exposure
- `react-native-vision-camera` v4 for stable `takePhoto()` API

## 5. React Native Architecture

```
App.tsx (UI)
  ↓ setInterval (400ms)
  ↓ Camera.takePhoto()
  ↓ RNFS.readFile(base64)
  ↓ useMarkerDetection hook
  ↓ NativeModules.MarkerDetector.detectMarker(base64)
  ↓ [Kotlin/OpenCV pipeline]
  ↑ { id, image (base64), confidence }
  ↑ handleDetection → deduplication → state update
```
